# 基础信息录入脚本

## 开场白

```
我来帮你创建这个剧本杀角色的 Skill。只需要回答 4 个问题，最后一个可以跳过。
准备好了就开始吧。
```

---

## 问题序列

### Q1：剧本杀名字

```
这个剧本叫什么名字？

例：归途七万里 / 你好，旧时光 / 年轮 / 古木吟
```

- 必填，不可跳过
- 接受任意字符串
- 用于生成 slug 的第一部分

---

### Q2：想生成的角色姓名 + 角色类型

```
你想蒸馏哪个角色？ta 是玩家位还是 NPC？

例：张眷信，玩家位
例：林伯，NPC
```

- 姓名必填
- 角色类型必填：玩家位 / NPC
- 角色类型影响后续材料要求：
  - 玩家位：需要提供该角色的玩家本
  - NPC：需要提供该 NPC 的人物小传（可能在 DM 手册中）

---

### Q3：使用者自己代入的角色姓名

```
你自己扮演的是哪个角色？（这决定了 ta 以什么身份和态度跟你聊天）

例：容葵
例：苏晚
```

- 必填，不可跳过
- 这个信息决定了生成 Skill 的「对话关系」——角色会以对使用者角色的态度来互动
- 提示：「如果你提供了自己角色的玩家本，角色会更了解你们之间的故事」

---

### Q4：补充信息（可选）

```
用一句话补充一些信息——剧本类型、时代背景、你们的关系、你对 ta 的印象，
想到什么写什么，跳过也行。

例：情感本 民国 恋人 他很温柔但有不得已的秘密
例：硬核本 现代 朋友 表面开朗实际城府很深
例：情感本 古代 师徒 她为了保护我牺牲了很多
```

从用户的回答中识别并提取以下字段（缺失的留空）：

- **剧本类型**：情感本 / 硬核本 / 机制本 / 恐怖本 / 欢乐本 / 其他
- **时代背景**：现代 / 民国 / 古代 / 架空 / 未来 / 其他
- **关系类型**：恋人 / 暗恋 / 亲人 / 朋友 / 仇人 / 师徒 / 搭档 / 其他
- **主观印象**：无法归类的自由描述，直接保留原文

#### 剧本类型说明

| 类型 | 特点 | 对 Persona 的影响 |
|------|------|-------------------|
| 情感本 | 重情感线、角色关系深 | 情感表达层加厚，互动细腻 |
| 硬核本 | 重逻辑推理 | 角色理性面突出，推理时的表达方式 |
| 机制本 | 重游戏机制 | 角色在博弈/选择时的决策模式 |
| 恐怖本 | 重氛围 | 角色面对恐惧的反应模式 |
| 欢乐本 | 轻松搞笑 | 表达风格更活泼、语气更轻松 |

#### 关系类型标签

关系类型对生成 Skill 的影响极大，决定了角色对使用者说话的方式：

| 关系 | 对话风格示例 |
|------|-------------|
| 恋人 | 亲密、有专属称呼、情感丰沛、有小脾气和撒娇 |
| 暗恋 | 欲言又止、故作镇定、偶尔脸红的细节 |
| 亲人 | 关心但不善表达、责备背后是担忧 |
| 朋友 | 轻松、互损、默契、仗义 |
| 仇人 | 冷淡、讽刺、但可能有复杂的隐藏情感 |
| 师徒 | 严厉中有慈爱、期望与失望交织 |
| 搭档 | 默契、信任、关键时刻的背靠背 |

---

## 确认汇总

收集完毕后展示：

```
确认一下信息：

🎭 剧本：{剧本杀名字}
👤 角色：{角色姓名}（{玩家位/NPC}）
🧑 你扮演的：{使用者角色姓名}
📎 补充：{类型} · {时代} · {关系} · {印象}

信息正确吗？确认后我们开始导入材料。
```

用户确认后进入 Step 2 材料导入。

---

## Slug 生成规则

基于收集到的信息生成 slug：

```
{剧本杀名字}_{角色姓名}_to_{使用者角色姓名}
```

- 中文保留原文
- 空格替换为下划线 `_`
- 示例：
  - `归途七万里_张眷信_to_容葵`
  - `你好旧时光_林晓_to_苏晚`
  - `年轮_老陈_to_小雨`
