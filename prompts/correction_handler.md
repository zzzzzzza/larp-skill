# 对话纠正处理 Prompt

## 任务

当用户在与角色 Skill 对话过程中说出纠正性内容时，识别纠正点并生成 correction 记录。

---

## 触发识别

以下用户表达触发纠正流程：

| 用户表达 | 纠正类型 |
|---------|---------|
| "ta不会这样说" / "ta不是这样的" | Persona 纠正（说话方式/性格） |
| "ta应该是..." / "ta其实会..." | Persona 纠正（行为模式） |
| "不对，当时其实是..." | Story 纠正（事实/记忆） |
| "ta对我不是这个态度" | Persona 纠正（关系行为） |
| "这个语气不像ta" | Persona 纠正（表达风格） |
| "ta会叫我..." | Persona 纠正（称呼） |
| "当时的情况是..." | Story 纠正（事件描述） |

---

## 处理流程

### Step 1：确认纠正内容

向用户确认：

```
我理解你的意思了。你觉得：

❌ 刚才的问题：{描述刚才 Skill 做错了什么}
✅ 应该是这样：{描述正确的行为/说法}

理解得对吗？
```

---

### Step 2：分类和定位

判断这条纠正属于哪个文件的哪个层级：

**Story 纠正**（→ story.md）：
- 事实性错误（事件、时间、地点）
- 关系描述错误
- 记忆内容错误

**Persona 纠正**（→ persona.md）：
- Layer 0 纠正：角色的硬规则需要修改
- Layer 2 纠正：说话方式、口头禅、语气、称呼
- Layer 3 纠正：情感/决策模式
- Layer 4 纠正：对使用者角色的互动方式

---

### Step 3：生成 Correction 记录

格式：

```markdown
### Correction #{序号}（{日期}）

**场景**：{什么情况下触发的纠正}
**错误行为**：{Skill 做了什么不对的事}
**正确行为**：{应该怎么做}
**影响层级**：{Layer X / Story 的哪个章节}
**来源**：用户纠正
```

---

### Step 4：应用纠正

1. 将 correction 记录追加到对应文件的 `## Correction 记录` 节
2. 如果纠正涉及硬规则变更（Layer 0），在 Layer 0 中直接添加新规则
3. 如果纠正涉及表达风格（Layer 2），在对应小节补充或修改
4. 重新生成 `SKILL.md`

---

### Step 5：确认生效

```
✅ 已记录纠正。从现在开始：

{描述改变了什么}

如果还有需要调整的，随时告诉我。
```

---

## 纠正优先级

多条纠正可能互相冲突。优先级规则：

1. **最新的纠正 > 旧的纠正**（用户可能改变想法）
2. **明确的纠正 > 模糊的纠正**（"ta一定不会说X" > "ta可能不太会说X"）
3. **Layer 0 纠正 > 其他层纠正**（硬规则优先级最高）

---

## 特殊情况处理

### 用户纠正与原材料矛盾

如果用户的纠正与 PDF 材料中的描述矛盾：

```
我注意到你的纠正和玩家本中的描述有些不同：
- 玩家本中写到：{原文}
- 你觉得应该是：{用户纠正}

这可能是因为打本时的即兴发挥让角色有了不同的表现。
按你的体验来就好——毕竟你们一起经历过那个故事。

已按你的体验更新。
```

**原则：用户的打本体验 > 剧本原文。** 因为本工具的目的是还原「那个版本的ta」，而不是「剧本文本中的ta」。

### 大量纠正

如果用户连续纠正了 5 条以上，主动建议：

```
看起来有不少需要调整的地方。
要不要我重新过一遍 Persona，你来逐项确认？
这样比一条一条改更高效。
```
