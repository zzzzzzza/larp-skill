# 增量 Merge Prompt

## 任务

你将收到：
1. 现有的 `story.md` 内容
2. 现有的 `persona.md` 内容
3. 新的原材料内容（PDF 文件或口述文字）

你的任务是判断新内容应该更新哪个部分，并输出增量更新内容。

**原则：只追加增量，不覆盖已有结论。如有冲突，输出冲突提示让用户决定。**

---

## Step 1：分类判断

将新内容中的每条信息归类：

| 信息类型 | 归入 |
|---------|------|
| 人物背景、经历、事件 | → story.md |
| 关系描述、共同记忆 | → story.md |
| 秘密、动机、目标 | → story.md |
| 时间线补充 | → story.md |
| 说话方式、口头禅、语气 | → persona.md |
| 性格特征、行为模式 | → persona.md |
| 情感表达、决策模式 | → persona.md |
| 对使用者角色的态度变化 | → persona.md |
| 两者都有 | → 分别归入 |

---

## Step 2：检查冲突

对比新内容与现有内容：

- 如果新内容**补充**了现有信息（增加了新细节）→ 直接追加
- 如果新内容**确认**了现有信息 → 忽略（不重复写）
- 如果新内容**与现有信息矛盾** → 输出冲突提示：

```
⚠️ 发现冲突：
- 现有：{现有描述}
- 新发现：{新内容描述}
- 来源：{文件名/类型}

可能的原因：
  [A] 不同角色视角看到的同一事件（多角色叙事差异）
  [B] 新材料修正了旧信息
  [C] 用户打本体验与剧本文本不同（即兴发挥）

建议：[保留现有 / 更新为新内容 / 两者都保留并标注视角]
请用户决定。
```

### 剧本杀特有的冲突处理

由于剧本杀的特殊性，同一事件在不同角色的玩家本中可能有不同描述。处理原则：

1. **目标角色的玩家本 > 其他角色的玩家本**：角色自己视角的描述优先
2. **DM 手册 > 所有玩家本**：DM 手册中的"真相"优先于任何角色的主观叙述
3. **用户口述 > PDF 文字**：用户的打本体验可能包含即兴发挥，属于"这个版本的故事"
4. **视角差异不是矛盾**：不同角色看同一事件的不同描述应该同时保留

---

## Step 3：生成更新 Patch

对 `story.md` 的更新，输出格式：
```
=== story.md 更新 ===

[追加到"关键事件时间线"节]
- {新事件}

[追加到"与{使用者角色}的专属记忆/共同经历"节]
- {新记忆}

[追加到"秘密档案"节]
- {新秘密}

[无更新] 或 [以上章节有更新]
```

对 `persona.md` 的更新，输出格式：
```
=== persona.md 更新 ===

[追加到"Layer 2/口头禅与高频词"节]
- 新口头禅：「{xxx}」

[追加到"Layer 4/对{使用者角色}"节]
- {新行为描述}

[无更新] 或 [以上章节有更新]
```

---

## Step 4：生成更新摘要

向用户展示：
```
本次更新摘要：
- story.md：追加了 {N} 条新信息（{简要描述}）
- persona.md：追加了 {N} 条新信息（{简要描述}）
- 发现 {N} 处冲突，需要你确认（见上方）

版本将从 {vN} 升级到 {vN+1}。
确认应用更新？
```
