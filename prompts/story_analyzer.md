# Story Memory 提取 Prompt

## 任务

你将收到：
1. 目标角色的玩家本 / 人物小传内容
2. 使用者角色的玩家本内容（可能有）
3. DM 手册内容（可能有）
4. 其他角色的玩家本内容（可能有）
5. 用户的口述补充（可能有）

你的任务是从这些材料中提取目标角色的**故事记忆**（Story Memory），重点关注该角色与使用者角色之间的关系和互动。

---

## 提取维度

### 1. 人物背景

```
① 基础身份
   - 姓名、年龄、性别
   - 社会身份/职业/地位
   - 时代背景（如果是古代/民国等非现代设定）

② 成长经历
   - 出身家庭
   - 关键成长事件
   - 塑造性格的经历

③ 核心秘密
   - 该角色自己知道但不会轻易说的事
   - 该角色自己都不知道的真相（如果 DM 手册中有）
   - 注意：区分「角色知道的」和「只有读者/DM 知道的」
```

### 2. 关键事件时间线

按时间顺序提取故事中的关键事件，标注：
- 事件名称
- 时间节点（具体或相对）
- 涉及的角色
- 该事件对目标角色的影响
- 该事件对目标角色与使用者角色关系的影响

格式：
```
[时间] 事件名称
  - 发生了什么
  - 谁参与了
  - 对角色的影响
  - 对关系的影响（如果涉及使用者角色）
```

### 3. 关系图谱

提取目标角色与所有其他角色的关系：

```
角色名 → 关系描述
  - 表面关系：{表面上的关系}
  - 真实关系：{实际的情感/利益关系}
  - 关键事件：{定义这段关系的事件}
  - 态度：{角色对此人的真实态度}
```

**重点标注**：目标角色与使用者角色的关系，需要更详细的提取：
- 关系的起源和发展轨迹
- 关键转折点
- 两人之间的共同记忆
- 未说出口的情感
- 矛盾与和解
- 对彼此的称呼（在不同时期/情境下是否不同）

### 4. 情感线

提取所有情感线索：

```
情感类型：爱情 / 友情 / 亲情 / 仇恨 / 愧疚 / 感恩 / 嫉妒 / ...
对象：{另一个角色}
发展轨迹：{从什么到什么}
关键表现：{具体的行为或对话}
结局状态：{故事结束时的状态}
```

### 5. 角色动机与目标

```
核心动机：角色做一切事情的根本驱动力
表面目标：角色表现出来的目标
隐藏目标：角色真正追求的东西
为了目标愿意付出的代价
为了目标绝不愿意牺牲的东西
```

### 6. 与使用者角色的专属记忆

这是最重要的部分——提取两个角色之间独特的互动记忆：

```
共同经历：{一起经历的事件}
日常互动：{日常相处的模式}
专属称呼：{彼此怎么称呼}
共同的秘密：{只有两人知道的事}
矛盾与冲突：{产生过的冲突}
温馨时刻：{温暖的记忆}
遗憾与未完成：{没来得及说的话、没来得及做的事}
最后的互动：{故事中最后一次互动的场景}
```

## 输出格式

将提取结果按以下结构输出，供 `story_builder.md` 使用：

```markdown
# {角色姓名} — Story Memory

## 基础身份
...

## 成长经历
...

## 关键事件时间线
...

## 关系图谱
### 与{使用者角色}的关系（核心）
...
### 与{其他角色1}的关系
...

## 情感线
...

## 动机与目标
...

## 与{使用者角色}的专属记忆
...

## 秘密档案
- 角色知道的秘密：...
- 角色不知道的真相：...（仅在有 DM 手册时提取，标注"角色不自知"）
```

---

## 注意事项

1. **信息来源标注**：标注每条信息来自哪个文件（目标角色玩家本/使用者玩家本/DM手册/其他）
2. **视角区分**：同一事件在不同角色眼中可能有不同版本，保留这些差异
3. **秘密分级**：区分「角色自己知道的秘密」和「只有 DM/读者知道的」
4. **矛盾保留**：不同材料中的矛盾信息不要合并，保留两个版本并标注来源
5. **补充推断**：如果口述补充了 PDF 中没有的信息，标注为「用户补充」
