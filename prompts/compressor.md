# 剧本文本压缩 Prompt

## 任务

你将收到一个从剧本杀 PDF 中提取的文本文件。你的任务是将其**压缩为原文的 1/3 ~ 1/2 长度**，同时保留所有对角色蒸馏有价值的信息。

压缩后的文本将用于后续的 Story Memory 和 Persona 分析，因此必须保留足够的细节来还原角色。

---

## 压缩原则

### 必须保留（一字不改地保留原文）

1. **角色对话原文**：所有引号内的对话，尤其是目标角色和使用者角色之间的对话
2. **角色称呼**：角色之间怎么互称（昵称、正式称呼、特殊称呼）
3. **口头禅和标志性台词**：反复出现的表达、角色的经典语句
4. **情感高潮段落**：表白、争吵、告别、重逢、牺牲等关键情感场景的**完整描写**
5. **时间节点和年龄**：具体的年份、年龄、时间跨度
6. **秘密和真相**：角色的秘密、隐藏的真相、不为人知的背景
7. **信物/道具描写**：对故事有象征意义的物品

### 可以压缩（改写为简洁摘要）

1. **环境描写**：景色、天气、建筑等纯环境段落 → 压缩为一句话
2. **过渡段落**：连接场景的叙述文字 → 删除或压缩为 `[场景转换：从X到Y]`
3. **重复信息**：多次提到的相同事件 → 只保留最详细的一次
4. **背景铺垫**：历史背景、时代说明等 → 压缩为关键信息
5. **次要角色互动**：与目标角色/使用者角色无关的其他角色对话 → 概括为一句

### 可以删除

1. **纯装饰性文字**：分隔线、页码、PDF 解析元信息
2. **重复的格式性内容**：章节标题重复、无内容的结构标记
3. **与目标角色完全无关的段落**：既没提到目标角色，也没提到使用者角色的内容

---

## 输出格式

```markdown
# {文件名} — 压缩版
# 目标角色：{角色名}
# 压缩比：约 {X}%

## {原始章节/段落标题}

{压缩后的内容}

## {下一个章节}

{压缩后的内容}
```

## 压缩示例

**原文**（120字）：
> 火车缓缓驶入站台，蒸汽弥漫在清晨的薄雾中。站台上挤满了送行的人，有人在哭泣，有人在叮嘱。张眷信站在人群中，手里紧紧攥着那张船票，心中百感交集。他转过身，看到容葵站在不远处，穿着一身素白的棉裙，眼眶微红。她轻声说："贾克斯，你一定要回来。"他看着她，喉咙发紧，只说了一句："等我。"

**压缩后**（75字）：
> 站台送行。张眷信攥着船票，看到容葵穿素白棉裙、眼眶微红。她轻声说："贾克斯，你一定要回来。"他喉咙发紧："等我。"

**保留了**：对话原文、角色称呼（贾克斯）、情感细节（眼眶微红、喉咙发紧）
**删除了**：环境描写（蒸汽、薄雾）、无关路人描写

---

## 注意事项

1. **不要编造**：压缩是提炼，不是改写。不要添加原文中没有的信息
2. **保持时间顺序**：压缩后的内容应保持原文的时间线顺序
3. **标注来源章节**：保留或标注内容来自哪个章节，方便后续追溯
4. **对话零损失**：目标角色和使用者角色的对话必须原文保留，绝不压缩
5. **宁多勿少**：如果不确定某段内容是否重要，保留它
