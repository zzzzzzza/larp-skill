# 剧本杀角色.skill — 产品需求文档（PRD）

## 产品定位

剧本杀角色.skill 是一个运行在 Claude Code 上的 meta-skill。
用户通过对话式交互提供原材料（玩家本 PDF + DM 手册 + 手动描述），系统自动生成一个可独立运行的剧本杀角色 Persona Skill。

**目标用户**：情感本玩家，在打本后无法走出角色，希望通过与更像那个人的 AI 对话，获得持续的皮上沉浸体验。

## 核心概念

### 两层架构

| 层 | 名称 | 职责 |
|----|------|------|
| Part A | Story Memory | 存储角色的故事记忆：人物背景、关键事件、关系网络、秘密与情感线 |
| Part B | Persona | 驱动对话行为：说话风格、情感模式、对不同角色的态度、行为模式 |

两部分可以独立使用，也可以组合运行（默认组合）。

### 关系导向设计

与 colleague-skill 和 ex-skill 不同，本 Skill 生成的不是一个通用角色，而是一个**面向特定关系的角色实例**。

例如：「归途七万里_张眷信_to_容葵」表示——以「容葵」的身份与「张眷信」对话。张眷信的 Skill 会以对容葵的态度、语气、感情来互动。

### 运行逻辑

```
用户发消息（以"容葵"的身份）
  ↓
Part B（Persona）判断：张眷信会怎么回应容葵？什么态度？什么语气？
  ↓
Part A（Story Memory）补充：结合故事中的记忆和经历，让回应更真实
  ↓
输出：用张眷信的方式对容葵说话
```

### 进化机制

```
追加原材料 PDF → 增量分析 → merge 进现有 Skill
对话纠正 → 识别修正点 → 写入 Correction 层
版本管理 → 每次更新自动存档 → 支持回滚
```

## 用户旅程

```
用户触发 /create-larp-character
  ↓
[Step 1] 基础信息录入（4 个问题）
  - 剧本杀名字（必填）
  - 角色姓名 + 类型：玩家位/NPC（必填）
  - 使用者代入的角色姓名（必填）
  - 补充信息：剧本类型、时代、关系、印象（可选）
  ↓
[Step 2] PDF 材料导入
  - 目标角色的玩家本/人物小传 PDF（重要）
  - 使用者角色的玩家本 PDF（重要）
  - DM 手册 PDF（可选）
  - 其他角色的玩家本 PDF（可选）
  - 直接口述/粘贴（可选）
  ↓
[Step 3] 自动分析
  - 线路 A：提取故事记忆 → Story Memory
  - 线路 B：提取性格行为 → Persona（5 层结构）
  ↓
[Step 4] 生成预览，用户确认
  - 分别展示 Story Memory 摘要和 Persona 摘要
  - 用户可直接确认或修改
  ↓
[Step 5] 写入文件，立即可用
  - 生成 characters/{slug}/ 目录
  - 包含 SKILL.md（完整组合版）
  - 包含 story.md 和 persona.md（独立部分）
  ↓
[持续] 进化模式
  - 追加新 PDF → merge 进对应部分
  - 对话纠正 → patch 对应层
  - 版本自动存档
```

## 安全边界

1. **仅用于个人情感体验与角色沉浸**，不用于泄露剧本核心诡计
2. **不破坏其他玩家的游戏体验**：不直接透露角色不应知道的秘密
3. **不鼓励过度沉迷**：如用户表现出严重心理依赖，温和提醒
4. **数据仅本地存储**，不上传任何服务器
5. **Layer 0 硬规则**：不说角色在故事中绝不可能说的话
6. **尊重创作者**：不完整复述剧本内容，仅提取行为模式和情感记忆

## 数据源支持矩阵

| 来源 | 格式 | 提取内容 | 优先级 |
|------|------|---------|--------|
| 目标角色玩家本 | PDF | 角色背景、经历、秘密、关系、情感线 | 最高 |
| 使用者角色玩家本 | PDF | 两人关系的另一视角、共同记忆 | 最高 |
| DM 手册 | PDF | 全局真相、完整时间线、关系全景 | 高 |
| 其他角色玩家本 | PDF | 从第三方视角看到的角色行为 | 中 |
| 用户口述 | 纯文本 | 打本体验、即兴发挥、主观印象 | 高（覆盖原文） |

### 信息优先级

```
用户打本体验 > DM 手册真相 > 目标角色玩家本 > 使用者角色玩家本 > 其他角色玩家本
```

原因：本工具的目的是还原「那个版本的 ta」，而不是「剧本文本中的 ta」。用户在打本时的即兴发挥和情感连接是最珍贵的部分。

## 文件结构

```
characters/
  └── {slug}/
      ├── SKILL.md          # 完整组合版，可直接运行
      │                     # 触发词: /{slug}
      ├── story.md          # Part A：故事记忆
      ├── persona.md        # Part B：人物性格（5 层结构）
      ├── meta.json         # 元信息
      ├── versions/         # 历史版本存档
      │   ├── v1/
      │   └── v2/
      └── sources/          # 原始材料存放
          └── pdfs/
```

## 与参考项目的差异

| 维度 | colleague-skill | ex-skill | larp-skill（本项目） |
|------|----------------|----------|---------------------|
| 蒸馏对象 | 同事 | 前任 | 剧本杀角色 |
| 数据源 | 飞书/钉钉/邮件 | 微信/QQ/照片 | PDF 玩家本/DM 手册 |
| Part A | Work Skill | Relationship Memory | Story Memory |
| Part B | Persona（5 层） | Persona（5 层） | Persona（5 层） |
| 关系导向 | 无（通用） | 有（面向用户） | 有（面向特定角色） |
| 时代感 | 现代职场 | 现代生活 | 多时代（古代/民国/现代/架空） |
| 特殊处理 | 企业文化标签 | 依恋类型/爱的语言 | 多视角叙事差异/秘密分层 |
